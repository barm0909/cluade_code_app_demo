# 倉庫管理機能

在庫ロットを複数の倉庫（販売倉庫・保留倉庫・不良倉庫など）に分けて管理できる機能です。

## デフォルト倉庫

初回起動時に以下の3倉庫が自動作成されます。

| 倉庫名 | 用途例 |
|--------|--------|
| 販売倉庫 | 通常販売用の在庫 |
| 保留倉庫 | 検品中・一時保管の在庫 |
| 不良倉庫 | 破損・期限切れなど販売不可の在庫 |

倉庫はlocalStorageに永続化され、追加・編集・削除が可能です（ロットが参照中の倉庫は削除不可）。

---

## 画面でできること

### 1. ロット一覧で倉庫を確認

商品行を展開するとロット一覧が表示され、各ロットの格納倉庫がカラーバッジで示されます。

```
ロットNo   賞味期限   倉庫          在庫数   操作
20260703   3日後      🟢 販売倉庫    10      [移動][編集][削除]
20260707   2026-07-07 🟠 保留倉庫     5      [移動][編集][削除]
```

### 2. 倉庫フィルターで絞り込み

検索バー横の「全倉庫」セレクトで倉庫を選ぶと、その倉庫のロットを持つ商品だけが一覧に表示されます。

### 3. ロット追加時に格納先を選択

「+ ロット追加」モーダルに倉庫セレクトが追加されました。省略した場合はデフォルトの販売倉庫になります。

### 4. 「移動」ボタンで倉庫間移動

各ロット行の「移動」ボタンから移動先倉庫と数量を指定して移動できます。

- **全量移動**: ロットの倉庫が更新されます（ロットIDは変わりません）
- **部分移動**: 元ロットの数量が減り、移動先に新しいロットが自動生成されます

### 5. 入出庫帳票への記録

移動操作は帳票に出庫・入庫の2件として自動記録されます。倉庫列に移動経路（例: `販売倉庫 → 不良倉庫`）が表示されます。

## 実装詳細

### 型定義

```typescript
interface Warehouse {
  id: string;
  name: string;
  color: string;
}

interface Lot {
  id: string;
  lotNo: string;
  expiryDate?: string;
  quantity: number;
  warehouseId: string; // 省略時は DEFAULT_WAREHOUSE_ID
}

type TransactionType = '入庫' | '出庫' | '移動';
```

### useInventory が返す新関数

| 関数 | 説明 |
|------|------|
| `addWarehouse(name, color)` | 倉庫を追加する |
| `updateWarehouse(id, name, color)` | 倉庫名・カラーを更新する |
| `deleteWarehouse(id)` | 倉庫を削除する（ロット参照中は失敗） |
| `moveLot(productId, lotId, targetWarehouseId, quantity)` | ロットを別倉庫へ移動する |

### ユーティリティ関数

```typescript
// 指定倉庫のロット合計在庫数を返す
totalQuantityByWarehouse(product: Product, warehouseId: string): number
```

### 後方互換

`warehouseId` を持たない既存ロットはロード時にデフォルト倉庫（`wh-sales`）へ自動移行します。
